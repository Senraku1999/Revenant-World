## 一、破折号 `——`

**国标用法：**

1. 标示注释补充或插入语
2. 标示总结上文或提示下文
3. 标示话题转换
4. 标示声音延长
5. 标示话语中断或间隔
6. 用于副标题之前

**项目规则：**

| 国标用法 | 项目处理 | 说明 |
|----------|----------|------|
| 1. 标示注释补充或插入语 | **收紧** | 引号外一律禁止，改逗号或冒号 |
| 2. 标示总结上文或提示下文 | **收紧** | 引号外一律禁止，改逗号或冒号 |
| 3. 标示话题转换 | **收紧** | 引号外一律禁止，改逗号或冒号 |
| 4. 标示声音延长 | 保留 | `"起雾了——"` 引号内合法 |
| 5. 标示话语中断或间隔 | 保留 | `"我不在了——"` 引号内合法 |
| 6. 用于副标题之前 | **收紧** | 项目无此场景 |

### 英文 Em Dash `—`（U+2014）

**独立规则**：JSON 非对话字段中，`—`（U+2014 em dash）为违禁字符，`——` 仅限对话引号内。`-`（U+002D）为唯一合法连字符，用于英文复合词与术语复合结构（half-open、灵力回路-切分、色彩阶-猩红迷雾）。

| 违规模式 | 替换 |
|----------|------|
| 连接解释说明 | 改 `:` 或 `. ` |
| 连接从句 | 改 `, ` 或 `. ` |
| 包裹同位语 | 删除 |

对话字段（`char_dialogue_examples[].response`）中的中文 `——` 不受此条限制，按中文破折号规则处理。

## 二、省略号 `……`

**国标用法：**

1. 标示引文的省略
2. 标示列举或重复词语的省略
3. 标示语意未尽
4. 标示说话时断断续续
5. 标示对话中的沉默不语

**项目规则：**

| 国标用法 | 项目处理 | 说明 |
|----------|----------|------|
| 1. 标示引文的省略 | **收紧** | 引号外一律禁止 |
| 2. 标示列举或重复词语的省略 | **收紧** | 引号外一律禁止 |
| 3. 标示语意未尽 | 保留 | 仅限引号内 |
| 4. 标示说话时断断续续 | 保留 | `"我绝不会放弃……绝不能放弃。"` 引号内合法 |
| 5. 标示对话中的沉默不语 | 保留 | `"安静地……消失吧。"` 引号内合法 |

> **"引号内"定义**：任何被 `""` 双引号包裹的文本，无论所在字段是 dialogue_examples、quirks、origin、relationships 还是简介/开场白的叙述段落。判断 `——` `……` 是否违规的唯一标准是：它在引号内还是引号外。引号内合法，引号外违规。

> **禁止**：将英文版 JSON 的标点选择作为中文版修改依据；将单字符 `…`（U+2026）作为 `……` 的替代方案——国标省略号形式为 `……`（六点、两字位），U+2026 不是标准中文省略号。

## 三、分号 `；`

**国标用法：**

1. 复句内部并列分句之间的停顿
2. 非并列关系（转折、因果）的多重复句，第一层前后两部分之间
3. 分行列举的各项之间

**项目规则：**

| 国标用法 | 项目处理 | 说明 |
|----------|----------|------|
| 1. 复句内部并列分句之间的停顿 | 保留 | 通过互换测试验证：`月：冷酷；果断；保护欲强；算计 / 明：怯懦；勤勉；无意识；温和` |
| 2. 非并列多重复句第一层前后之间 | **收紧** | 因果、递进、转折、时序均改逗号或句号 |
| 3. 分行列举的各项之间 | 保留 | 列举项本质是平行对等项 |

**互换测试：** 分号前后互换，语义不变 → 并列，分号正确。语义改变 → 非并列，改逗号或句号。

**与国标的关系：项目规则是国标的真子集（proper subset）。**

## 四、引号

**国标用法：**

1. 标示直接引用
2. 标示着重论述或强调
3. 标示特殊含义（别称、简称、反语等）

**项目规则：**

| 国标用法 | 项目处理 | 说明 |
|----------|----------|------|
| 1. 直接引用 | 保留 | 功能不变 |
| 2. 着重论述或强调 | 保留 | 功能不变 |
| 3. 特殊含义 | 保留 | 功能不变 |
| 引号形式 | **收紧** | 仅允许 `"`（U+0022 英文直双引号）；国标中文引号 `“”`（U+201C/U+201D）禁用 |

---

## 五、感叹号 `！` 与问号 `？`

**国标用法：**

1. `！`：感叹句末、语气强烈的祈使句和反问句
2. `？`：疑问句末、选择问句末

**项目规则：**

| 国标用法 | 项目处理 | 说明 |
|----------|----------|------|
| 对话内 | 保留 | `"准备好吗！"` `"你要去哪？"` |
| 叙述中 | **收紧** | 叙述段落中禁止使用 |

## 六、叙述中标点总表

| 符号 | 国标允许叙述中 | 项目允许叙述中 | 变化 |
|------|:---:|:---:|------|
| `。` `，` `、` | ✓ | ✓ | 一致 |
| `；` | ✓（并列+非并列多重复句） | 仅并列 | 收紧 |
| `——` | ✓（注释/转换/总结） | ✗ | 收紧 |
| `……` | ✓（省略/未尽） | ✗ | 收紧 |
| `！` `？` | ✓ | ✗ | 收紧 |
| 引号 | ✓（国标形式 `“”` U+201C/U+201D） | ✓（仅 `""` U+0022） | 收紧（形式替换） |
| `：` | ✓（提示下文/注释说明） | 仅限字段标签与子标签 | 收紧 |
| `—`（U+2014） | —（英文标点） | ✗（仅限对话字段） | 新增 |

> **冒号「叙述」与「结构化描述」的边界**：`combat_style`、`weapon`、`char_special_abilities` 等字段为结构化元数据，其内 `名称：说明` 的冒号属于信息架构中的子标签，不视为叙述标点，合规。`overview`、`origin`、`mission`、`personality` 子字段、`char_relationships` 内容等为叙事散文，其内禁止冒号——适用「叙述仅三个标点 ，。、」。

## 七、句号 vs 逗号：项目判例

逗号表示话未说完，句号表示一个完整意思结束。以下三种情况应用逗号而非句号：

**1. 名词性标签后接展开**：`A。B。` 中 A 是身份/角色/能力标签而无谓语动词时，A 与 B 之间应用逗号。如 `晨昏事务所的防御专家，性格温和。`（非 `专家。性格温和。`）

**2. 连续动作/事件共享主语**：同一主语的行为链中，句号切断动作连贯性。如 `持桂妮薇儿冲入战场，在决斗中轮换五德。`（非 `战场。在决斗中`）

**3. 平行标签对举**：`X：…。Y：…` 中 X 与 Y 为同一维度下平行对等项时，应用分号连接。如 `月：冷酷；果断；保护欲强；算计 / 明：怯懦；勤勉；无意识；温和`

能力描述中"概要→机制/效果"的展开、武器描述中"武器名→性能特征"的展开、起源中连续人生阶段的衔接，均适用以上规则。
