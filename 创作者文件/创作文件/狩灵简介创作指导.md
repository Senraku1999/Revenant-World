# 狩灵世界观 · 简介创作指导

## 一、简介的核心定位

简介是角色卡的**结构化精简版**，用于快速建立角色认知。它与角色卡 JSON 和开场白共同构成角色的"三位一体"表达体系：

- **JSON 角色卡**：AI 理解角色的核心数据源（见 `创作者文件/创作文件/狩灵角色卡创作指导.md`）
- **开场白**：用户与角色的"第一印象"（见 `创作者文件/创作文件/狩灵开场白创作指导.md`）
- **简介**：人类阅读的角色概览，从 JSON 中提取关键信息并按标准格式组织

## 二、标准格式：五段式结构

简介采用 `创作者文件/创作文件/简介标准格式.md` 定义的五段式结构，此处不重复列出，请直接查阅该文件。

### 2.1 各段填写规则

| 大项 | 必填 | 内容来源 | 注意事项 |
|------|------|----------|----------|
| 一、基础信息 | 是 | JSON char_persona + char_faction + char_status + char_rank | 特征 = 发型发色 + 瞳色 + 着装风格。评级纯中文格式，见2.4 |
| 二、基础能力 | 是 | JSON char_basic_abilities | 三维度缺一不可，使用等级名 + 中文说明 |
| 三、特殊能力 | 条件 | JSON char_special_abilities | **若无内容，删除整个大项，后续序号自动补充** |
| 四、装备记录 | 条件 | JSON char_persona.weapon | **若无内容，删除整个大项，后续序号自动补充** |
| 五、综合评估 | 是 | JSON char_description + char_personality | 用评估方视角写，署名格式固定 |

### 2.2 格式规则

文件结构与空行规则见 `创作者文件/创作文件/简介标准格式.md`，以该文件为准。三、四若无内容则删除整个大项并自动补充序号；文件以空行开头，不以换行符结尾。

标点遵循 GB/T 15834 中文标准语法。简介字段标签中的 `：` 为格式定义，不可替换。

### 2.3 评估方署名与文体规则

详见 `创作者文件/创作文件/简介标准格式.md` 综合评估部分的评估方规则。核心原则：评估方统一使用组织名，不精确到个人。补充说明：协会官方下属事务所所长的评估方为对接科室，事务所成员的评估方为事务所。

**文体由评估方决定。** 简介是评估方按标准格式撰写的内部归档档案，综合评估段落的文体风格须与评估方的身份和正式程度匹配。形式度光谱（从高到低）：

| 形式度 | 评估方 | 文体要求 |
|--------|--------|----------|
| 最高 | 狩灵协会1科（总务科） | 严格公文。第三人称客观陈述，禁止文学比喻、口语调侃、第二人称叙述 |
| 高 | 狩灵协会2-6科 | 公文为主。允许适度实操化措辞（如3科"老家伙"），但整体保持专业判断口吻 |
| 中 | 事务所所长（官方型，如贝尔金） | 实地报告风格。口语程度介于科室和千乐之间 |
| 低 | 事务所所长（亲切型，如千乐） | 允许适度亲切。千乐由薰（6科科长、其兄长）撰写，可在公文框架内流露个人视角 |
| 豁免 | 千乐亲笔（如水镜） | 完全个人化。非协会评估方，允许口语、调侃、直接引语 |

判断标准：
- **文学化描写**（比喻、意象、抒情语言）、**口语调侃**（"吓唬同事""老家伙"）、**第二人称叙述**（"你不知道…"）→ 仅低形式度或豁免级可接受
- **主观评价**（"令人惊叹""恶趣味颇重"）→ 与评估方级别成反比，1科最严格
- **感情渲染**（"很多很多人"）→ 1科禁止，其他科酌情

### 2.4 评级格式规范

**简介中评级必须使用纯中文格式。** 不同角色类别格式如下：

| 角色类别 | 简介评级（纯中文） |
|----------|-------------------|
| 狩灵（1-5阶） | `1阶` ~ `5阶` |
| 狩灵（色彩阶） | `色彩阶-称号`（如 `色彩阶-赤红正义`） |
| 游魂 | `无` |
| 罪灵 | `传闻级` / `怪谈级` / `梦魇级` / `灾厄级` / `终焉级` |
| 异常/扭曲 | `灾厄级` / `终焉级` 等 |

## 四、常见问题

### 4.1 结构问题

| 问题 | 原因 | 修正 |
|------|------|------|
| 大项序号断裂 | 删除空大项后未调整序号 | 删除后自动补充，如五→四 |
| 大项间空行不对 | 不了解格式规范 | 大项间仅保留**一个空行** |
| 保留了"武器：无" | 误解"删除整个大项"规则 | 删除整个"四、装备记录"大项 |

### 4.2 内容问题

| 问题 | 原因 | 修正 |
|------|------|------|
| 信息与 JSON 不一致 | 简介和 JSON 分别维护 | 简介信息必须从 JSON 提取 |
| 评估方署名错误 | 不了解署名规则 | 参照 2.3 节评估方规则。协会官方下属事务所评估方为对接科室（如晨昏事务所→2科），非固定 1科 |
| 特征描述过于简略 | 未使用 JSON 中的完整信息 | 特征 = hair + face + clothing（features 有则追加） |
| 一般称呼误用 alias | 不了解一般称呼规则 | 参照 `创作者文件/创作文件/简介标准格式.md` 中的三条规则 |
| 评级与 JSON 不一致 | 简介和 JSON 分别维护 | 评级（char_rank）必须与 JSON 严格一致 |
| 特例能力套用标准等级 | 强制为自定义能力贴等级标签 | 与 JSON 保持一致，JSON 未用标准等级则简介也不标 |
| 回路产物误列为装备 | 将能力延伸物当作独立武器 | 回路产物（如锁链、灵力针）已在三中描述，不重复列入四 |
| 关系对象使用全名 | 不了解一般称呼规则 | 关系对象必须用一般称呼（如"千乐"），禁用全名（如"花坂 千乐"） |
| 评级格式不统一 | 不了解各角色类别评级格式 | 参照 2.4 节评级格式规范：狩灵=`X阶`/`色彩阶-称号`，游魂=`无`，罪灵/异常=`X级` |
| 身份称谓不规范 | 不了解协会/事务所的身份称谓区别 | 协会科室成员写"科员"，事务所/工坊/锈钟成员写"成员"，不使用花名 |
