## 档案头规则

角色/实体卡简介在文件开头（空行之前）添加档案分类标签，独占一行，与正文之间保留一个空行。事件卡、世界观卡不需档案头。

| 角色类别 | 档案头 |
|----------|--------|
| 狩灵 | `狩灵档案：` |
| 游魂 | `游魂档案：` |
| 罪灵 | `罪灵档案：` |
| 异常 | `异常档案：` |
| 扭曲 | `扭曲档案：` |

## 格式规则

1. 文件开头为档案头（独占一行），其下保留**一个空行**，然后接正文
2. 文件末尾最后一行后**不保留空行**
3. 大节之间保留**一个空行**
4. 同节内子项之间不留空行，内容连续排列
5. 无子项的大节（如三仅有灵力回路一行），标题下一行直接写内容，内容后保留一个空行接下一节
6. 综合评估正文与下方 `评估方` 行之间保留**一个空行**
7. 标点符号遵循 GB/T 15834 中文标准语法，项目附加规则见 CLAUDE.md 标点铁律
