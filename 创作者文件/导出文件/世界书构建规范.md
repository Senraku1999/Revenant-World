# 世界书构建规范

## 概述

世界书（World Info / Lorebook）用于 AI 运行时注入世界观知识和角色速查。本项目世界书采用**基础层 + 阵营层**双层架构，基础层提供核心规则，阵营层按分组引入角色词条。

---

## 数据读取规则

> **TAG 池是所有条目的 keys 唯一来源。** 构建世界书时，角色条目和基础规则条目的 keys 都必须从 `TAG池.md` 对应条目读取，禁止硬编码空数组或自造 keys。同类条目必须走同一读取路径。

---

## 层级结构

```
基础层（7 条核心规则，每本世界书必含）
        │
        ├── 狩灵.json（基础规则 + 无阵营角色 + 第一次扭曲）
        ├── 狩灵 花坂家.json（基础规则 + 花坂家 + 花坂家宴）
        ├── 狩灵 晨昏事务所.json（基础规则 + 晨昏4人）
        ├── 狩灵 沃拉瑟斯.json（基础规则 + 钢翼菲利普 + 沃拉瑟斯）
        ├── 狩灵 来生事务所.json（基础规则 + 来生2人）
        ├── 狩灵 追猎.json（基础规则 + 追猎3人）
        └── 狩灵 全角色.json（基础规则 + 协会1科 + 新宿站）
```

> **子集只做加法，不裁剪基础规则。** 阵营层世界书在基础规则之上追加角色词条，不从基础层删除任何条目。

## 阵营层：角色词条通用参数

所有角色词条采用统一参数：

| 参数 | 值 |
|------|-----|
| `selective` | `true` |
| `insertion_order` | `90` |
| `depth` | `4` |
| `probability` | `100` |

**角色词条内容来源：** `创作者文件/导出文件/角色关系网/{分组}/{角色}.json` 全文序列化为字符串，写入 `entry.content`。若角色无对应分组子目录，则取 `全角色/{角色}.json`。角色关系网是世界书词条内容的**唯一真相源**，世界书不做独立编辑。

**角色词条 keys 来源：** `创作者文件/导出文件/TAG池.md` 为 keys 的**唯一真相源**。构建世界书时，每个角色的 `entry.keys` 必须从触发词表中对应角色的 Tag 列逐条读取，禁止从 `char_fullname` 或 `char_alias` 自动拆词生成。触发词表已经过人工审核，执行了短词覆盖长词去重、亲属称谓仅阵营书、色彩称号/帮会职位入 tag 等规则，自动拆词无法复现这些决策。

## 构建流程

1. 创建基础层条目（7 条），内容从 `创作者文件/导出文件/world info/` 读取
2. 按阵营分组，从 `创作者文件/导出文件/角色关系网/{分组}/` 读取角色词条 JSON
3. 每个角色词条：将 `创作者文件/导出文件/角色关系网/{分组}/{角色}.json` 全文序列化为 `entry.content`
4. 基础规则条目的 `constant` 在各阵营世界书中保持一致
5. 最终输出为 SillyTavern 兼容的 World Info JSON 格式
