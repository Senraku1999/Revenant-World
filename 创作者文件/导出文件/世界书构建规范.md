# 世界书构建规范

## 概述

世界书（World Info / Lorebook）用于 AI 运行时注入世界观知识和角色速查。本项目世界书采用**基础层 + 阵营层**双层架构，基础层提供核心规则，阵营层按分组引入角色词条。

---

## 数据读取规则

> **TAG 池是所有触发词条目的 keys 唯一来源。** 构建世界书时，角色条目与非常驻基础条目的 keys 都必须从 `TAG池.md` 对应分组读取，禁止硬编码或自造 keys；常驻条目（`constant: true`）keys 为 `[]`，无需触发词。同类条目必须走同一读取路径。

---

## 层级结构

```
基础层（8 条核心规则，每本世界书必含）
        │
        ├── 狩灵.json（仅基础规则，零角色词条）
        ├── 狩灵 全角色.json（基础规则 + 全部角色词条 + 第一次扭曲）
        ├── 狩灵 四色音.json（基础规则 + 四色音成员）
        ├── 狩灵 晨昏事务所.json（基础规则 + 晨昏成员）
        ├── 狩灵 沃拉瑟斯.json（基础规则 + 沃拉瑟斯成员）
        ├── 狩灵 花坂家.json（基础规则 + 花坂家成员）
        ├── 狩灵 来生事务所.json（基础规则 + 来生成员）
        └── 狩灵 追猎.json（基础规则 + 追猎成员）
```

> **子集只做加法，不裁剪基础规则。** 阵营层世界书在基础规则之上追加角色词条，不从基础层删除任何条目。

## 阵营层：角色词条通用参数

所有角色词条采用统一参数：

| 参数 | 值 |
|------|-----|
| `constant` | `false` |
| `selective` | `false` |
| `insertion_order` | `0` |
| `depth` | `4` |

**角色词条内容来源：** `创作者文件/导出文件/关系网/全角色/{角色}.json` 全文序列化为字符串，写入 `entry.content`（关系网管线第一步的全量产物；各阵营子目录为派生副本，供分组浏览，不作构建输入）。关系网是世界书词条内容的**唯一真相源**，世界书不做独立编辑。

**角色词条 keys 来源：** `创作者文件/导出文件/TAG池.md` 为 keys 的**唯一真相源**。构建世界书时，每个角色的 `entry.keys` 必须从 TAG 池对应分组的 Tag 列逐条读取，禁止从 `char_fullname` 或 `char_alias` 自动拆词生成。TAG 池已经过人工审核，执行了短词覆盖长词去重、亲属称谓仅阵营书、色彩称号/帮会职位入 tag 等规则，自动拆词无法复现这些决策。

---

## 阵营层世界书覆盖清单

成员分配的唯一真相源为 `导出世界书.ts` 的 `WORLDBOOK_BUILD` 表，本节不复录名单（避免漂移）。要点：

- **狩灵.json**：仅 8 条基础规则，零角色词条。铁律。
- **狩灵 全角色.json**：基础规则 + 全部角色词条 + 第一次扭曲。keys 从 TAG 池 `狩灵 全角色` 分组取。
- **各阵营册**（四色音 / 晨昏事务所 / 沃拉瑟斯 / 花坂家 / 来生事务所 / 追猎）：基础规则 + `WORLDBOOK_BUILD` 对应成员词条。keys 从 TAG 池各自阵营分组取。
- 阵营册成员与关系网阵营目录（`FACTION_MEMBERS`）、PNG 导出分配（`PNG_EXPORT_ASSIGN`）三表相互独立，不必一致。

---

## 构建流程

1. 创建基础层条目（8 条），内容从 `创作者文件/导出文件/world info/` 读取
2. 按 `WORLDBOOK_BUILD` 分组，从 `创作者文件/导出文件/关系网/全角色/` 读取角色词条 JSON
3. 每个角色词条：将 `关系网/全角色/{角色}.json` 全文序列化为 `entry.content`
4. 基础规则条目的 `constant` 在各阵营世界书中保持一致
5. 最终输出为 SillyTavern 兼容的 World Info JSON 格式
