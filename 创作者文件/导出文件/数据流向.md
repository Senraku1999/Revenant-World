# 数据流向

## 两条独立管线

本项目有两条互不交叉的数据管线：

---

## 管线一：角色卡导出

```
{角色}.json + {角色}简介.md + {角色}开场白.md
        │
        └── 拼装 ──→ {角色}角色卡.json（SillyTavern 导入用）
```

**说明：** 三个源文件直接拼成一张角色卡，不做任何内容修改，仅组装。详见 `角色卡导出规范.md`。

---

## 管线二：世界书构建

```
创作者文件/导出文件/world info/*.md   创作者文件/导出文件/角色关系网/{分组}/{角色}.json
  （基础规则条目）                      （角色补充词条）
        │                                    │
        └──────────┬─────────────────────────┘
                   │
                   ▼
              完整世界书 JSON
         （基础规则 + 角色词条）
```

**说明：**

- **`创作者文件/导出文件/world info/`** 是世界书的**主体**，提供 7 条基础规则（宏观规则、世界观、灵能者、评级、异常、狩灵协会、锈钟）。这些条目每本世界书都必须包含，子集只做加法不裁剪。
- **`创作者文件/导出文件/角色关系网/`** 是世界书的**补充条目**，按阵营将角色摘要作为 selective 词条插入，丰富世界书的角色知识覆盖。
- 世界书 = world info 打底 + 角色关系网补角色。角色关系网不是世界书的唯一来源，只是角色词条的来源。

---

## 角色关系网的来源

```
{角色}.json（角色卡）
      │
      └── 提取扁平化 ──→ 创作者文件/导出文件/角色关系网/{分组}/{角色}.json
```

角色关系网是角色卡 JSON 的扁平化摘要，服务于世界书的角色词条需求。
