# 数据流向

## 三条串联管线

`npm run 全管线` = 管线一 → 管线二 → 管线三 依次执行。

---

## 管线一：角色卡 → 关系网（`导出关系网.ts`）

```
角色卡/**/{card_name}.json（事件卡、世界观卡除外）
        │
        ├── 精简（stripToRelationFormat）──→ 关系网/全角色/{card_name}.json（全量）
        │
        └── 按 FACTION_MEMBERS 分配 ──→ 关系网/{阵营}/{card_name}.json（派生副本，供分组浏览）
```

---

## 管线二：关系网 → 世界书（`导出世界书.ts`）

```
world info/*.md（8 条基础规则）      关系网/全角色/{角色}.json（按 WORLDBOOK_BUILD 取成员）
        │                                    │
        └──────────┬─────────────────────────┘
                   ▼
        世界书/狩灵*.json（8 册：基础规则 + 角色词条，keys 来自 TAG池.md）
```

---

## 管线三：底图 + 世界书 → PNG（`导出角色卡.ts`）

```
{card_name}.json + {card_name}简介.md + {card_name}开场白.md + 世界书（按 PNG_EXPORT_ASSIGN 分配）
        │
        └── 拼装 ST 卡（compact JSON）──→ base64 嵌入 底图/{card_name}.png ──→ 导出角色卡/{card_name}.png
```

---

详见 `角色卡导出规范.md`（管线三与铁律）、`世界书构建规范.md`（管线二）。三张分配表（`FACTION_MEMBERS` / `WORLDBOOK_BUILD` / `PNG_EXPORT_ASSIGN`）相互独立，唯一真相源在各自脚本内。
