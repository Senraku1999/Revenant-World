# 角色卡导出规范

## 概述

每个角色由三个源文件组成，最终拼装为一份 SillyTavern 兼容的 `chara_card_v2` 格式角色卡 JSON，供 AI 角色扮演平台导入使用。

---

## 源文件

| 文件 | 内容 | 导出用途 |
|------|------|----------|
| `{角色}.json` | 英文版角色数据 | 序列化为字符串写入 `data.description` |
| `{角色}开场白.md` | 初见场景 | 原文写入 `data.first_mes` |
| `{角色}简介.md` | 狩灵协会 4 科档案格式简介 | 原文写入 `data.creator_notes` |
---

## 输出文件结构（SillyTavern chara_card_v2）

```json
{
  "spec": "chara_card_v2",
  "spec_version": "2.0",
  "data": {
    "name": "狩灵 · {char_name}",
    "description": "{角色}.json 全文序列化为字符串",
    "first_mes": "{角色}开场白.md 原文",
    "creator_notes": "{角色}简介.md 原文",
    "character_book": "按阵营嵌入对应成品世界书全部条目（见下方分配表）",
    "tags": ["狩灵", "大世界", "世界观", "现代", "超自然", "角色扮演", "sfw"],
    "creator": "千乐",
    "character_version": "1.0",
    "extensions": {}
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `spec` | string | 固定值 `"chara_card_v2"` |
| `spec_version` | string | 固定值 `"2.0"` |
| `data.name` | string | 格式 `"狩灵 · {char_name}"`，`{char_name}` 取角色英文名 |
| `data.description` | string | `{角色}.json` 全文序列化为字符串（保留原有 JSON 结构，不做扁平化） |
| `data.first_mes` | string | `{角色}开场白.md` 原文，不做任何修改 |
| `data.creator_notes` | string | `{角色}简介.md` 原文，不做任何修改 |
| `data.character_book` | object[] | 角色所属阵营的成品世界书全部条目，格式见下方世界书分配表 |
| `data.tags` | string[] | 固定 7 个标签，顺序不可变 |
| `data.creator` | string | 固定值 `"千乐"` |
| `data.character_version` | string | 固定值 `"1.0"` |
| `data.extensions` | object | 固定空对象 `{}` |

---

## 固定字段锁定规则

以下字段在所有角色卡中**永远相同**，不受角色差异影响，导入/导出时不得修改：

| 字段 | 锁定值 |
|------|--------|
| `spec` | `"chara_card_v2"` |
| `spec_version` | `"2.0"` |
| `data.creator` | `"千乐"` |
| `data.tags` | `["狩灵", "大世界", "世界观", "现代", "超自然", "角色扮演", "sfw"]` |
| `data.character_version` | `"1.0"` |
| `data.extensions` | `{}` |

---

## 世界书分配表

每张角色卡按其所属阵营嵌入对应成品世界书。世界书文件位于 `创作者文件/导出文件/成品世界书/`。

| 成品世界书 | 条目数 | 分配给 |
|-----------|:--:|------|
| `狩灵.json` | 7（仅宏观） | 协会狩灵/锈钟/工坊/游魂/罪灵/事件卡 |
| `狩灵 全角色.json` | 57（7宏观+50角色） | 新宿站、协会1科 |
| `狩灵 四色音.json` | 11（7宏观+4角色） | 心音、花音、弦音、铃音 |
| `狩灵 晨昏事务所.json` | 11（7宏观+4角色） | 贝尔金、贝拉、弗洛伦、菲利普 |
| `狩灵 沃拉瑟斯.json` | 9（7宏观+2角色） | 沃拉瑟斯、菲利普 · 钢翼 |
| `狩灵 花坂家.json` | 10（7宏观+3角色） | 薰、千乐、百合子 |
| `狩灵 来生事务所.json` | 9（7宏观+2角色） | 爱、星流 |
| `狩灵 追猎.json` | 10（7宏观+3角色） | 慎、劫、烬 |

### character_book 格式

`data.character_book` 为对象，含 `name`（世界书名）和 `entries`（条目数组）：

```json
{
  "name": "狩灵 晨昏事务所",
  "entries": [
    {
      "id": 0,
      "keys": [],
      "secondary_keys": [],
      "comment": "宏观规则",
      "content": "...",
      "constant": true,
      "selective": true,
      "insertion_order": 100,
      "enabled": true,
      "position": "before_char",
      "use_regex": false,
      "extensions": {}
    },
    {
      "id": 1,
      "keys": ["贝尔金", "黎明", "晨昏"],
      "secondary_keys": [],
      "comment": "贝尔金",
      "content": "{...关系网JSON...}",
      "constant": false,
      "selective": true,
      "insertion_order": 90,
      "enabled": true,
      "position": "before_char",
      "use_regex": false,
      "extensions": {}
    }
  ]
}
```

- `id`：从 0 递增的序号
- `keys`：触发词。角色条目取 `全角色触发词表.md` 中 Tag，宏观条目为空数组
- `secondary_keys`：备用触发词，始终 `[]`
- `selective`：**始终 `true`**（允许用户手动开关条目）
- `use_regex`：始终 `false`（精确匹配，非正则）
- `name`：取世界书文件名，如 `狩灵`、`狩灵 四色音`、`狩灵 晨昏事务所`

---

## 导出流程

1. 读取 `{角色}.json`，将全文序列化为字符串
2. 读取 `{角色}开场白.md`，保留原文
3. 读取 `{角色}简介.md`，保留原文
4. 读取角色对应的成品世界书，全部条目转为 character_book 格式
5. 按上述结构拼装为 V2 JSON，**所有 JSON 序列化统一使用 `\r\n` 换行、2 空格缩进**（`JSON.stringify(obj, null, 2).replace(/\n/g, '\r\n')`）
6. Base64 编码后写入 PNG 的 tEXt/chara chunk
