# 角色卡导出规范

## 源文件

| 文件 | 导出用途 |
|------|----------|
| `{角色}.json`（英文） | → `description` |
| `{角色}开场白.md` | → `first_mes` |
| `{角色}简介.md` | → `creatorcomment` |

---

## 输出格式

SillyTavern 实际读取的是**扁平结构**——所有核心字段在顶层，同时 `data` 对象内镜像一份。

```json
{
  "name": "狩灵 · {char_name}",
  "description": "{英文 JSON 全文，\\r\\n 换行}",
  "personality": "",
  "scenario": "",
  "first_mes": "{开场白原文，\\r\\n 换行}",
  "mes_example": "",
  "creatorcomment": "{简介原文，\\r\\n 换行}",
  "avatar": "none",
  "chat": "狩灵 · {char_name} - {导出时间戳}",
  "talkativeness": "0.5",
  "fav": false,
  "tags": ["狩灵", "现代", "超自然", "角色扮演", "战斗", "sfw"],
  "spec": "chara_card_v2",
  "spec_version": "2.0",
  "create_date": "{导出时间戳}",
  "data": {
    "name": "{同上}",
    "description": "{同上}",
    "personality": "",
    "scenario": "",
    "first_mes": "{同上}",
    "mes_example": "",
    "creator_notes": "{简介原文，\\r\\n 换行}",
    "system_prompt": "",
    "post_history_instructions": "",
    "tags": ["狩灵", "现代", "超自然", "角色扮演", "战斗", "sfw"],
    "creator": "千乐",
    "character_version": "",
    "alternate_greetings": [],
    "extensions": {
      "talkativeness": "0.5",
      "fav": false,
      "world": "{世界书名}",
      "depth_prompt": {
        "prompt": "",
        "depth": 4,
        "role": "system"
      }
    },
    "character_book": {
      "name": "{世界书名}",
      "entries": "{世界书全部条目}"
    }
  }
}
```

### 关键规则

| 规则 | 说明 |
|------|------|
| `\r\n` | description / first_mes / creatorcomment 内部字符串的换行符，**不是外层 JSON 的换行符** |
| 扁平 + 镜像 | 顶层和 `data` 内同时存在 name/description/first_mes 等字段，值相同 |
| `creatorcomment` vs `creator_notes` | 顶层用 `creatorcomment`，`data` 内用 `creator_notes` |
| `chat` | 格式 `"狩灵 · {char_name} - {M/D/YYYY @HHh MMm SSs SSSms}"` |
| `character_version` | **空字符串** `""`，不是 `"1.0"` |
| `avatar` | 固定 `"none"` |
| `extensions.world` | 世界书文件名（不含路径和扩展名） |

---

## 固定字段

| 字段 | 值 |
|------|-----|
| `spec` | `"chara_card_v2"` |
| `spec_version` | `"2.0"` |
| `tags` | `["狩灵", "现代", "超自然", "角色扮演", "战斗", "sfw"]` |
| `creator` | `"千乐"` |
| `talkativeness` | `"0.5"` |
| `fav` | `false` |
| `personality` / `scenario` / `mes_example` / `system_prompt` / `post_history_instructions` | `""` |
| `alternate_greetings` | `[]` |

## 世界书条目格式

每条记录必须包含以下全部字段。缺任何一项导致 SillyTavern 导入失败。

```json
{
  "id": 0,
  "keys": [],
  "secondary_keys": [],
  "comment": "条目名",
  "content": "...",
  "constant": true,
  "selective": true,
  "insertion_order": 100,
  "enabled": true,
  "position": "after_char",
  "use_regex": true,
  "extensions": {
    "position": 1,
    "exclude_recursion": false,
    "display_index": 0,
    "probability": 100,
    "useProbability": true,
    "depth": 4,
    "selectiveLogic": 0,
    "group": "",
    "group_override": false,
    "group_weight": 100,
    "prevent_recursion": false,
    "delay_until_recursion": false,
    "scan_depth": null,
    "match_whole_words": null,
    "use_group_scoring": false,
    "case_sensitive": null,
    "automation_id": "",
    "role": 0,
    "vectorized": false,
    "sticky": 0,
    "cooldown": 0,
    "delay": 0
  }
}
```

| 字段 | 值 |
|------|-----|
| `selective` | 始终 `true` |
| `use_regex` | 始终 `true` |
| `enabled` | 始终 `true` |
| `position` | 宏观条目（keys=[]）→ `"after_char"`；角色条目（keys 非空）→ `"before_char"` |
| `secondary_keys` | 始终 `[]` |
| `extensions` | 必须包含上表全部字段，不可为空 `{}` |

## 导出流程

1. 读取 `{角色}.json` → JSON 序列化为字符串 → `description`（内部 `\r\n`）
2. 读取 `{角色}开场白.md` → `first_mes`（内部 `\r\n`）
3. 读取 `{角色}简介.md` → `creatorcomment` / `data.creator_notes`（内部 `\r\n`）
4. 读取角色对应成品世界书 → `data.character_book`（`{"name": "...", "entries": [...]}`）
5. 拼装为上述扁平 + 镜像结构
6. **JSON 序列化必须用 compact 格式**：`json.dumps(card, separators=(',', ':'))`——无缩进、无空格。`\r\n` 仅存在于内层字符串字段（description / first_mes / creatorcomment），外层 JSON 没有任何换行
7. PNG：将 compact JSON base64 编码，嵌入 PNG 的 tEXt/chara chunk（替换已有 chara chunk，或在 IEND 前插入）
