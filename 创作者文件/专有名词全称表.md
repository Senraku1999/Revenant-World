# 专有名词全称表

> 所有 JSON 字段（char_faction、char_status、char_relationships、char_description、char_background 等）中出现的专有名词必须使用本表"强制全称"列。禁止使用"简称/变体"列中的任何形式。
>
> 审查脚本 `审查脚本_自动检查.py` 读取本表进行精确匹配检测。
>
> **匹配模式**：`子串` = 文本中任何位置出现即命中；`独立词` = 仅当简称前后无中文字符（不被包含在更大的中文复合词中）时命中，避免"事务所"误伤"绯月事务所"。

## 扩展规则

以下为跨字段通用规则，非逐词替换：

| 规则 | 说明 |
|------|------|
| 狩灵协会X科 | 任何位置出现，不得省略"狩灵协会"前缀，不得用英文 department X 替代 |
| 组织名中英一致 | char_faction 使用中文全称，char_status / char_relationships 等英文描述字段中同义组织也必须用中文全称 |
| 评级后缀 | JSON char_rank 中罪灵/异常评级必须带"级"后缀（如 梦魇级），简介中评级匹配对应中文格式 |
