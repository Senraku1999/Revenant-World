# CLAUDE.md

## 项目概述

狩灵世界观是以当代日本双重社会为背景的灵能题材角色扮演世界观。每个角色含三个文件：`{角色名}.json`（角色卡）、`{角色名}简介.md`、`{角色名}开场白.md`。

| 目录 | 内容 |
|------|------|
| `创作者文件/创作文件/` | 创作指导、格式标准 |
| `创作者文件/审查文件/` | 审查规则、标点国标、自动检查脚本 |
| `创作者文件/导出文件/world info/` | 世界观与规则条目（World Info） |
| `创作者文件/导出文件/角色关系网/` | 按阵营分目录的角色关系副本 |
| `创作者文件/导出文件/成品世界书/` | 按阵营分册的 ST 世界书 JSON |
| `角色卡/` | 全部角色卡（按阵营分目录） |
| `导出角色卡/` | 导出角色卡 |
| `小说化内容/` | 世界观叙事文学 |

## 导出操作强制前置

> **此节优先级高于一切。违反即错误，无例外。**

执行以下任一操作前，第一步必须 Read `创作者文件/导出文件/角色卡导出规范.md`：导出角色卡 JSON、导出角色卡 PNG、更新/重建成品世界书、同步角色关系网副本。未读该文件直接动手 = 必然出错。禁止凭记忆执行。

**修改后导出管线**：角色文件修改定档后按顺序执行——① 同步角色关系网全部副本，② 重建成品世界书，③ 更新角色卡 PNG。世界书分配规则见 `创作者文件/导出文件/角色卡导出规范.md`。

**PNG 导出前置检查**：执行 PNG 导出前，必须先确认 `底图/` 中已存在对应角色的底图 PNG 文件。无底图不导出。

## 一、核心规则

**以卡为准**：动手前，先看三张现有卡。任何创作或修改，第一步不是读规范，而是打开至少三张同类角色卡的同字段，确认该字段的实际写法。规范描述模板，现有卡片才是落地标准。规范与全体卡片不一致时，以卡片为准。此条优先级高于一切角色文件修改。

**标点**：中文遵循 GB/T 15834，数字用法遵循 GB/T 15835，异形词遵循 GF 1001—2001。英文遵循英文标准语法。
- 角色卡（JSON/简介/开场白）：叙述仅 `，` `。` `、` 三标点，`""` 为唯一合法引号，`；` 仅限并列关系，`——` `……` 仅限对话内。嵌套引号（`""` 内 `''`）合法。
- 小说化内容：中国标准文学标点，中文引号 `""`，句尾 `？！` 解锁，`——` 和 `……` 在叙述中同样禁止（仅限对话内）。
- `：` 在枚举（`X：A、B、C`）、解释（`X：Y说明`）、引述（`说："..."`）中均为合法用法。

标点审查三级体系：一级 GB/T 15834 → 二级三步法 → 三级项目追加，详见 `创作者文件/审查文件/`。

**JSON**：遵循 RFC 8259。UTF-8 编码，无 BOM。中英文 JSON 同步修改，结构一致。
**JSON**：遵循 RFC 8259。UTF-8 编码，无 BOM。中英文 JSON 同步修改，结构一致。

**ControlBoundary**：允许试探发问，禁止明确设定玩家身份、目的或关系。

**共享事件同步**：修改涉及多个角色的共享事件后，必须逐文件检查全部关联角色，确认时间锚点、责任归属、基本事实无矛盾。

## 二、创作

> **每次修改角色文件之前，以下两步不得跳过：**

**第一步：重读规范（禁止依赖会话记忆）。** 根据修改类型，用 Read 打开对应指导文件：
**第一步：重读规范（禁止依赖会话记忆）。** 根据修改类型，用 Read 打开对应指导文件：

| 修改内容 | 必读文件 |
|----------|----------|
| JSON | `创作者文件/创作文件/狩灵角色卡创作指导.md`、`创作者文件/创作文件/角色卡标准格式.md` |
| 简介 | `创作者文件/创作文件/狩灵简介创作指导.md`、`创作者文件/创作文件/简介标准格式.md` |
| 开场白 | `创作者文件/创作文件/狩灵开场白创作指导.md` |
| 任何角色文件 | `创作者文件/导出文件/world info/宏观规则.md` |
| 导出/PNG/世界书/关系网 | `创作者文件/导出文件/角色卡导出规范.md` |
| appearance 字段 | `创作者文件/创作文件/Appearance字段格式指导.md` |
| 术语/专有名词 | `创作者文件/创作文件/专有名词全称表.md` |
| 基础能力字段 | `创作者文件/创作文件/基础能力等级示例.md` |

**第二步：修改后跑检查，全部达标再报告。** 运行 `自动检查.py`（命令见第四节）。任一检查未通过 → 继续修改 → 再跑检查 → 直到全部达标。不等用户提醒。
