## 一、核心规则

**标点**：中文遵循 GB/T 15834（`创作者文件/审查文件/一级审查/GB_T_15834_标点符号用法.md`），数字用法遵循 GB/T 15835（`创作者文件/审查文件/一级审查/GB_T_15835_数字用法.md`），异形词遵循 GF 1001—2001（`创作者文件/审查文件/一级审查/GF_1001_2001_异形词整理表.md`）。英文遵循英文标准语法。项目追加约束：叙述仅 `，` `。` `、` 三标点，`""` 为唯一合法引号，`；` 仅限并列关系，`——` `……` 仅限对话内。句号 vs 逗号判例详见 `创作者文件/审查文件/二级审查/标点符号二级审查.md`。审查采用三步法（删去→替换→重构），详见 `创作者文件/审查文件/三级审查/标点符号三级审查.md`。

**JSON**：遵循 RFC 8259。UTF-8 编码，无 BOM。

**ControlBoundary**：允许试探发问，禁止明确设定玩家身份、目的或关系。

其余创作规则（Token 预算、评级格式、外观描述、角色独立性等）均以 `创作者文件/创作文件/` 对应指导文件为准，此处不重复。其余已确认的判定边界详见 `创作者文件/审查文件/其他审查/宏观审查计划.md`。

## 二、创作

> **每次修改角色文件之前，以下两步不得跳过：**

**第一步：重读规范（禁止依赖会话记忆）。**

根据修改的文件类型，用 Read 工具打开对应指导文件过一遍，不做任何编辑，先读完：

| 修改内容 | 必读文件 |
|----------|----------|
| JSON | `创作者文件/创作文件/狩灵角色卡创作指导.md`、`创作者文件/创作文件/角色卡标准格式.md` |
| 简介 | `创作者文件/创作文件/狩灵简介创作指导.md`、`创作者文件/创作文件/简介标准格式.md` |
| 开场白 | `创作者文件/创作文件/狩灵开场白创作指导.md` |
| 任何角色文件 | `创作者文件/导出文件/world info/宏观规则.md` |

**第二步：修改后跑检查，全部达标再报告。** 运行 `审查脚本_自动检查.py`（命令见第四节）。任一检查未通过 → 继续修改 → 再跑检查 → 直到全部达标。不等用户提醒。

**中英文 JSON 同步**：修改任何英文字段时，必须同步修改对应的中文 JSON（`_zh.json`），反之亦然。审查流程须检查中英文 JSON 结构一致性（关系键、能力键、对话示例数量等）。

**分批审查流程**：每批修改执行后，更新候选文件排除已完成项。写候选文件前先读源文件确认句号仍在位，已改的不重复列出。用句号序号定位，不用关键词 hint 匹配。

## 三、团队

本项目配备三支 AI 团队，通过 Agent 工具调用。agent 配置文件位于 `.claude/agents/`。

### 项目经理

| Agent 文件 | 职责 |
|-----------|------|
| `pm-orchestrator` | 任务分解、agent 调度、进度追踪、结果汇总。复杂多步骤任务时优先调用。 |

### 学术团队（创作/审查阶段通用）

| Agent 文件 | 领域 |
|-----------|------|
| `anthropologist` | 文化系统、亲属关系、仪式、帮会亚文化、双重社会 |
| `geographer` | 城市分布、地盘空间、场景地理锚定、通勤逻辑 |
| `historian` | 时间线、事件因果、年代断代、组织起源 |
| `narratologist` | 叙事结构、开场白三段式、对话风格、角色 voice |
| `psychologist` | 人格张力、创伤处理、动机可信度、缺陷真实性 |

### 审查团队（审查阶段）

| Agent 文件 | 职责 |
|-----------|------|
| `review-coordinator` | 流程管理、任务分配、结果汇总、最终裁决。不逐行审查。 |
| `punctuation-reviewer` | 按 GB/T 15834 + 项目两条硬规则，逐字标点扫描。 |
| `logic-reviewer` | 先聚合再比对，跨文件/跨角色逻辑矛盾检测。 |

标点审查按三级体系执行（一级国标 → 二级三步法 → 三级项目追加），详见 `创作者文件/审查文件/`。全量审查流程：pm-orchestrator 调度 → 自动脚本 → 三级标点审查 → 逻辑审查并行 →（深度审查可选：学术五视角并行）→ 汇总分级 → 清理中间产物。

> **审查后清理**：审查结束后删除所有子 agent 产生的临时文件，包括：`*_output.txt`、`*_extract*.py`、`*_debug*`、`cross_file_*.py/.txt`、`historian_extract*`、`extract_*.py`、`.claude/scripts/` 下临时脚本、`创作者文件/审查文件/` 下次级目录中的一次性提取脚本与输出。仅保留 `审查脚本_自动检查.py`、`审查脚本_GBT15834_一级国标.py`、`secondary_punct_review.py` 三个永久脚本。

## 四、常用命令

```bash
python "创作者文件/审查文件/审查脚本_自动检查.py"                     # 全项目自动扫描（Token/JSON/MD标点/交叉验证/中英文一致性）
python "创作者文件/审查文件/一级审查/审查脚本_GBT15834_一级国标.py"       # 一级国标标点审查（句末/句内点号/引号配对/书写形式）
python "创作者文件/审查文件/secondary_punct_review.py"                # 二级三步法机械预筛（逗号删去/句号替换/分号互换）
python -m json.tool 文件.json > /dev/null                      # JSON 语法检查
```
