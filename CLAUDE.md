## 一、核心规则

**标点**：中文遵循 GB/T 15834，英文遵循英文标准语法。项目附加两条硬规则：`""`（英文直双引号）为唯一合法引号，`——` `……` 仅限对话（引号内）使用。

**JSON**：遵循 RFC 8259。UTF-8 编码，无 BOM。

**ControlBoundary**：允许试探发问，禁止明确设定玩家身份、目的或关系。

**Token**：角色卡四段式 A[300-500] BC[0-500] D[200-400] 总[500-1400]，开场白[900-1100]，tiktoken cl100k_base。

**评级格式**：JSON `色彩阶`，简介 `色彩阶-称号`，带连字符。游魂 JSON `"None"` ↔ 简介 `"无"`。

其余已确认的判定边界及细则详见 `审查计划.md`。

## 二、创作

创作前必须重读对应指导文件（禁止依赖会话记忆）：
- 通用：`世界书设定/宏观规则.md`
- JSON：`创作者文件/狩灵角色卡创作指导.md`、`创作者文件/角色卡标准格式.md`
- 简介：`创作者文件/狩灵简介创作指导.md`、`创作者文件/简介标准格式.md`
- 开场白：`创作者文件/狩灵开场白创作指导.md`

修改后运行：`python "创作者文件/审查脚本_自动检查.py"`

## 三、团队

本项目配备三支 AI 团队，通过 Agent 工具调用。agent 配置文件位于 `.claude/agents/`。

### 项目经理

| Agent 文件 | 职责 |
|-----------|------|
| `pm-orchestrator` | 任务分解、agent 调度、进度追踪、结果汇总。复杂多步骤任务时优先调用。 |

### 学术团队（创作/审查阶段通用）

| Agent 文件 | 领域 |
|-----------|------|
| `anthropologist` | 文化系统、亲属关系、仪式、帮会亚文化、双重社会 |
| `geographer` | 城市分布、地盘空间、场景地理锚定、通勤逻辑 |
| `historian` | 时间线、事件因果、年代断代、组织起源 |
| `narratologist` | 叙事结构、开场白三段式、对话风格、角色 voice |
| `psychologist` | 人格张力、创伤处理、动机可信度、缺陷真实性 |

### 审查团队（审查阶段）

| Agent 文件 | 职责 |
|-----------|------|
| `review-coordinator` | 流程管理、任务分配、结果汇总、最终裁决。不逐行审查。 |
| `punctuation-reviewer` | 按 GB/T 15834 + 项目两条硬规则，逐字标点扫描。 |
| `logic-reviewer` | 先聚合再比对，跨文件/跨角色逻辑矛盾检测。 |

审查按 `审查计划.md` 六步执行（读取规则 → 复述理解 → 基础格式层 → 逻辑一致性层 → 汇总分级 → 自查修正），详见 `审查方法论.md`。全量审查流程：pm-orchestrator 调度 → 自动脚本 → 标点审查 + 逻辑审查并行 →（深度审查可选：学术五视角并行）→ 汇总分级 → 自查。
